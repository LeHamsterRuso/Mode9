Por dónde empezar

Esta guía monta un juego en modo 9 por etapas, de lo más simple a lo más completo. Cada etapa añade una sola cosa nueva sobre la anterior, así que puedes pararte donde quieras y ya tendrás algo que funciona. Sigue el mismo orden que los ejemplos MODE9_01 a MODE9_05 que vienen con la DLL.

Lo mínimo que necesitas: un escenario en .OBJ (triangulado, Y hacia arriba) y su textura en .MAP. Si aún no tienes ninguno, usa los del ejemplo (CASTLE.OBJ + MAP/CASTLE.MAP). Para preparar tus propios modelos, mira Modelos OBJ.

Etapa 1 · Un mundo vacío por el que pasearte

El esqueleto de todo juego en modo 9. Cargas el escenario con LOAD_WLD9(), colocas la cámara con MODE9_SET() y arrancas el render con START_MODE9(). A partir de ahí, mover la cámara es como mover cualquier proceso de DIV: advance() para avanzar y cambiar angle para girar.

Lo esencial: 
PROGRAM paso1;
IMPORT "MODE9.DLL";
CONST
    c_m9 = 200;
GLOBAL
    STRUCT m9   z; camera; height; angle;   END
BEGIN
    set_mode(320200);
    set_fps(60, 0);

    LOAD_WLD9("CASTLE.OBJ", "MAP/CASTLE.MAP", 100);  // escenario
    LOAD_SKY9("MAP/SKY.MAP");                     // cielo (opcional)
    MODE9_STEP(70);                              // subir escalones

    MODE9_SET(70, 0);                            // altura del ojo, pitch
    START_MODE9(id, OFFSET m9.z, 0);         // ¡a renderizar!

    LOOP
        if (key(_w)) advance(20); end
        if (key(_s)) advance(-20); end
        if (key(_left))  angle += 4000; end
        if (key(_right)) angle -= 4000; end
        if (key(_esc)) break; end
        FRAME;
    END
    STOP_MODE9();
END

Ver: LOAD_WLD9() · START_MODE9() · MODE9_SET() · MODE9_STEP() · MODE9_COLLISION()

Etapa 2 · Poblarlo con billboards (sprites 2D)

Lo más rápido de meter en el mundo son los billboards: dibujos 2D planos que siempre miran a la cámara y se escalan con la distancia, como en el modo 7. Sirven para árboles, antorchas, objetos y hasta enemigos sencillos. La forma más cómoda es la automática: das al proceso ctype = c_m9 y sus coordenadas de mundo en x, y, z, y el modo 9 lo dibuja solo cada frame sin que llames a nada.

Un proceso billboard automático: 
// En el BEGIN principal, tras START_MODE9, carga el FPG y crea props:
load_fpg("FPG/MODE9.FPG");
antorcha(300, -500, 0);
antorcha(-300, -500, 0);
...
PROCESS antorcha(x, y, z)
BEGIN
    ctype = c_m9;          // <- lo marca como billboard del modo 9
    graph = 11;            // dibujo de la antorcha en el FPG
    size  = 100;           // tamaño base (la distancia hace el resto)
    LOOP
        // x, y, z son coordenadas de MUNDO; no toques nada más.
        FRAME;
    END
END

Ver: MODE9_BILLBOARD() · MODE9_CTYPE() · MODE9_OCCLUDE() · MODE9_PROJECT()

Etapa 3 · Meter objetos 3D de verdad

Cuando un billboard plano no basta (quieres rodear al objeto y verlo desde todos los lados), usa un modelo 3D. Se carga una vez con LOAD_OBJ9(), que devuelve un handle, y se dibuja cada frame con DRAW_OBJ9() en la posición que quieras. El mismo handle se puede dibujar varias veces para repetir el objeto.

Cargar y dibujar un modelo 3D: 
GLOBAL
    h_gato;
...
    h_gato = LOAD_OBJ9("OBJ/CAT.OBJ", "MAP/CAT.MAP", 50);
    LOOP
        // se dibuja en (x, y, z) mirando a un angle dado, cada frame
        DRAW_OBJ9(h_gato, 400, -1300, 0, 0);
        // ... mover la cámara, etc.
        FRAME;
    END
Carga en segundo plano: LOAD_OBJ9 devuelve el handle al instante pero el fichero tarda unos frames en estar listo. No pasa nada: DRAW_OBJ9 ignora el modelo hasta que carga, así que aparece solo. Si quieres esperar, usa MODE9_OBJ9_READY().

Ver: LOAD_OBJ9() · DRAW_OBJ9() · MODE9_OBJ9_READY()

Etapa 4 · Animar un objeto 3D

Para que el objeto se mueva (un gato que respira, un enemigo que anda), cárgalo con LOAD_OBJ9_ANIM() en vez de LOAD_OBJ9. La animación son varias poses guardadas como ficheros numerados (CAT0.OBJ, CAT1.OBJ...), y el modo 9 mezcla las poses intermedias por ti para que se vea fluido. Se dibuja con DRAW_OBJ9_ANIM(), que lleva un parámetro de más, fpos, que dice por qué punto de la animación va.

Cargar una animación y reproducirla en bucle: 
CONST
    NFRAMES = 10;      // CAT0.OBJ .. CAT9.OBJ
    LOOP_CS = 300;     // una vuelta dura 3 s (300 centesimas)
GLOBAL
    h_gato;
...
    h_gato = LOAD_OBJ9_ANIM("OBJ/CAT", "MAP/CAT.MAP", 50, NFRAMES);
    LOOP
        // fpos atado al reloj: igual de rapido vaya el juego a los FPS que vaya
        DRAW_OBJ9_ANIM(h_gato, 400, -1300, 0, 0,
                       (timer[0] % LOOP_CS) * (NFRAMES*256) / LOOP_CS);
        FRAME;
    END

Ver: LOAD_OBJ9_ANIM() · DRAW_OBJ9_ANIM()

Etapa 5 · Pasar de un mapa a otro

Un nivel grande se parte en varios mapas y se va cargando el siguiente al cruzar una puerta. La clave: LOAD_WLD9() recarga el escenario EN CALIENTE, sin parar el modo 9 ni perder la cámara. El truco para que no se note el salto es un fundido a negro mientras se cambia la malla y se recoloca al jugador.

Cruzar una puerta hacia el interior: 
// ¿El jugador está pegado a la puerta? (distancia al cuadrado < radio²)
if ((x-PUERTA_X)*(x-PUERTA_X) + (y-PUERTA_Y)*(y-PUERTA_Y) < RADIO2)
    fade_off(); frame; fade_on(); frame;   // fundido a negro

    signal(type props_patio, s_kill);     // quitar lo del mapa viejo

    LOAD_WLD9("OBJ/CASTIN.OBJ", "MAP/CASTIN.MAP", 100);  // nuevo mapa
    SKY9_OFF();                              // interior: sin cielo
    SET_ENV_COLOR9(8, 8, 12);           // ambiente oscuro

    x = 0; y = -300; angle = -90000;   // recolocar al jugador
end
Cuidado con re-disparar la puerta: tras un cambio, deja unos cuantos frames de margen (un "cooldown") antes de volver a comprobar puertas, o entrarás y saldrás sin parar. Y como LOAD_WLD9 es bloqueante, ten a mano ignore_error(142) por si salta el watchdog en hardware real.

Ver: LOAD_WLD9() · SKY9_OFF() · SET_ENV_COLOR9() · STOP_MODE9()

Y a partir de aquí, a pulir: niebla y distancia de visión (SET_FOG9(), MODE9_FARCLIP()), ángulo de cámara (MODE9_FOV()), pantalla partida para dos jugadores (MODE9_SPLIT()) o más velocidad en máquinas modestas (MODE9_DOWNSCALE()). Lo tienes todo en Funciones.
