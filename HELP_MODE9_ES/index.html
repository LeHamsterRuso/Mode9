El Modo 9 de DIV Games Studio 2

El modo 9 (la librería MODE9.DLL) es una ampliación de DIV Games Studio 2 para mostrar y recorrer un escenario en tres dimensiones que cargas desde un fichero .OBJ (el formato de malla que exportan programas como Blender). A diferencia de un fondo plano, el escenario tiene volumen real: paredes, suelos, techos y rampas con cualquier inclinación, y te puedes mover libremente por dentro de él.

Es el hermano mayor del modo 8: allí el mapa se construía con sectores planos (.WLD); aquí el mapa es una malla 3D cualquiera modelada en un programa de 3D. La cámara es un proceso normal de DIV que colocas donde quieras y apuntas en la dirección que quieras. Lo más habitual es ponerla en los ojos del jugador (vista en primera persona, como en los ejemplos), pero nada te obliga a ello: también sirve para una cámara que sigue al personaje por detrás, una vista cenital o cámaras fijas.

Sobre ese escenario 3D puedes dibujar tus personajes y objetos de dos maneras: como sprites planos (dibujos 2D que se escalan con la distancia y siempre miran a la cámara, igual que en el modo 7) o como modelos 3D de verdad, con animación suave.

Conceptos básicos

Mover la cámara no tiene ningún misterio: como es un proceso normal, la adelantas con advance(), la mueves de lado con xadvance(), la haces girar cambiando su campo angle y la subes o la bajas con la altura del ojo (m9.height). Lo que ves en pantalla es el mundo desde donde esté ese proceso.

Sistema de coordenadas del mundo (uds DIV2, divididas por el unitscale para pasar a unidades OBJ):

EjeSignificado
X = proceso.xPlano del suelo (este / oeste).
Z = proceso.yPlano del suelo (norte / sur).
Y = proceso.z + m9.heightEje vertical (altura, Y hacia arriba como en Blender / OBJ).
yaw = proceso.angleGiro horizontal, en milésimas de grado (180000 = 180 grados).
pitch = m9.angleInclinación vertical, en milésimas de grado (0 = horizonte).
Sobre m9 y c_m9: una DLL no puede crear STRUCTs ni c_types del lenguaje DIV2 (los genera el compilador). Por eso la estructura m9 (campos z, camera, height, angle, los mismos que m8) y la constante c_m9 = 200 se declaran en el PRG y se le pasan a la DLL con START_MODE9() y MODE9_SET(). Los procesos billboard usan ctype = c_m9.

Un programa mínimo

Programa de ejemplo: 
PROGRAM mode9_demo;
IMPORT "MODE9.DLL";
CONST
    c_m9 = 200;           // ctype lógico del billboard (debe valer igual en la DLL)
GLOBAL
    STRUCT m9             // misma forma que m8: z, camera, height, angle
        z; camera; height; angle;
    END
    scr_x; scr_y;
BEGIN
    set_mode(320200);
    set_fps(60, 0);
    scr_x = MODE9_SCRW();
    scr_y = MODE9_SCRH();
    load_fpg("FPG/MODE9.FPG");

    // 1) Cargar el escenario (malla + textura + escala) en una sola llamada
    LOAD_WLD9("CASTLE.OBJ", "MAP/CASTLE.MAP", 100);
    LOAD_SKY9("MAP/SKY.MAP");      // cielo panoramico
    MODE9_STEP(70);                // subir escaleras de hasta 70 uds

    // 2) Situar la cámara y arrancar el render 3D
    MODE9_SET(70, 0);              // altura del ojo y pitch
    START_MODE9(id, OFFSET m9.z, 0);

    // 3) Bucle de juego: mover la cámara
    LOOP
        if (key(_w)) advance(20);  end
        if (key(_s)) advance(-20); end
        angle += (scr_x/2 - mouse.x) * 200;   // girar con el ratón
        if (key(_esc)) STOP_MODE9(); break; end
        FRAME;
    END
END
