LOAD_WLD9()

LOAD_WLD9( <fichero OBJ> , <textura MAP> , <escala> )

Descripción:

Carga el escenario 3D en UNA sola llamada: la malla del fichero .OBJ, su textura .MAP y la escala (unidades DIV2 por unidad del OBJ). Es el equivalente a load_wld() del modo 8: sirve tanto para cargar el primer nivel como para CAMBIAR de nivel en caliente.

El escenario es una malla arbitraria (paredes, suelos y techos con cualquier inclinación) modelada por ejemplo en Blender y exportada como OBJ triangulado con Y hacia arriba. La textura de este parámetro es el material 0 y se aplica por defecto a todos los materiales; con MODE9_MATERIAL() puedes dar una textura distinta a cada usemtl.

Parámetros:

ParámetroSignificado
fichero OBJNombre del fichero .OBJ del escenario. Recuerda el límite de DOS: el nombre base debe tener 8 caracteres o menos, o la carga fallara en silencio.
textura MAPFichero .MAP del material 0. Si pones "" o "0", el escenario se dibuja con flat-shading (sombreado plano sin textura).
escalaUnidades DIV2 por cada unidad del OBJ (unitscale). 100 es lo habitual; el doble (200) hace el escenario el doble de grande. Un valor <=0 no cambia la escala actual.

Retorna: El número de triángulos cargados (0 si falla la carga).

Nota: La carga es BLOQUEANTE: con escenarios muy grandes puede saltar la excepción 142 ("el proceso parece bloqueado, esperando FRAME"). Si ocurre, puedes ignorarla con ignore_error(142).
Nota: Cambiar de mapa con LOAD_WLD9 NO requiere parar el modo 9: libera la malla y los materiales antiguos y reconstruye las colisiones, sin tocar la cámara.
Programa de ejemplo: 
// Cargar el patio del castillo (material 0 = CASTLE.MAP)
ntris = LOAD_WLD9("CASTLE.OBJ", "MAP/CASTLE.MAP", 100);
if (ntris == 0)
    say("No se pudo cargar el escenario");
end

// ... más tarde, al cruzar una puerta, cambiar de mapa EN CALIENTE:
LOAD_WLD9("CASTIN.OBJ", "MAP/CASTIN.MAP", 100);
x = 0; y = -300; angle = -90000;   // recolocar al jugador en el nuevo mapa

El valor de retorno (número de triángulos) sirve para comprobar que la carga ha ido bien: si es 0, el fichero no existe o no es válido.
