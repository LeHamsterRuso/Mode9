DRAW_OBJ9_ANIM()

DRAW_OBJ9_ANIM( <handle> , <x> , <y> , <z> , <angle> , <fpos> )

Description:

Draws an animated model (loaded with LOAD_OBJ9_ANIM()) in the world, just as DRAW_OBJ9() draws a still one. Call it every frame (before FRAME) for each copy of the model you want to see.

The only new thing is the last parameter, fpos, which tells how far into the animation the model is at that instant. Instead of counting whole poses, it counts in fine steps: each pose is 256 units. So:

fpos is......and you see
0pose 0 (the first .OBJ).
256pose 1.
512pose 2.
384exactly halfway between pose 1 and pose 2 (smooth blend).

The in-between values are what give the smoothness: mode 9 blends the two neighbouring poses depending on how close you are to each. The full run goes from 0 to nframes×256 and, once past the end, wraps back to the start on its own (continuous loop). You don't draw each pose one by one: just raise fpos little by little and the animation runs itself.

The most convenient and robust way to move fpos is to tie it to the clock (the timer) rather than to your own counter, so the cycle always lasts the same number of seconds whatever frame rate the game runs at (see the example).

Parameters:

ParameterMeaning
handleHandle returned by LOAD_OBJ9_ANIM.
x, y, zWorld position in DIV2 units.
angleOrientation in thousandths of a degree.
fposPoint in the animation, at 256 per pose (0 = pose 0, 256 = pose 1, in-between values = blend). Range 0..nframes×256, loops automatically.

Returns: 0.

Example program: 
CONST
    NFRAMES = 10;        // poses: CAT0.OBJ .. CAT9.OBJ
    LOOP_CS = 300;       // one cycle lasts 300 hundredths = 3 seconds
PRIVATE
    fpos;
END
    h_cat = LOAD_OBJ9_ANIM("CAT", "MAP/CAT.MAP", 100, NFRAMES);
    LOOP
        // fpos tied to the clock: the loop lasts LOOP_CS regardless of FPS.
        // timer[0] counts in hundredths of a second; spread the cycle
        // 0..NFRAMES*256 over LOOP_CS and repeat it with modulo (%).
        fpos = (timer[0] % LOOP_CS) * (NFRAMES * 256) / LOOP_CS;
        DRAW_OBJ9_ANIM(h_cat, 0, 500, 0, 0, fpos);
        FRAME;
    END

Because mode 9 blends the poses for you, a smooth animation needs very few files (ten poses are enough for a smooth walk cycle). To make it faster or slower, lower or raise LOOP_CS (the seconds one loop takes).

Tip: a model loaded with LOAD_OBJ9() (still, a single pose) can also be drawn with DRAW_OBJ9_ANIM: since it has only one pose, the DLL ignores fpos and always draws that one. So you can use the same command for everything.
For the curious (you don't need this): internally, fpos is split in two: the high part (fpos>>8, i.e. fpos÷256) picks the starting pose, and the low part (fpos&255, the remainder) says how far to blend towards the next one. That's why 256 = exactly one pose and 384 = one pose and a half.
