LOAD_WLD9()

LOAD_WLD9( <OBJ file> , <MAP texture> , <scale> )

Description:

Loads the 3D scene in ONE single call: the mesh from the .OBJ file, its .MAP texture and the scale (DIV2 units per OBJ unit). It is the equivalent of mode 8's load_wld(): it serves both for loading the first level and for HOT-SWAPPING levels at runtime.

The scene is an arbitrary mesh (walls, floors and ceilings at any angle) modelled, for example, in Blender and exported as a triangulated OBJ with Y up. The texture from this parameter is material 0 and is applied by default to all materials; you can assign a different texture to each usemtl with MODE9_MATERIAL().

Parameters:

ParameterMeaning
OBJ fileName of the scene's .OBJ file. Remember the DOS limit: the base name must be 8 characters or fewer, otherwise loading will fail silently.
MAP textureThe .MAP file for material 0. If you pass "" or "0", the scene is drawn with flat-shading (no texture).
scaleDIV2 units per OBJ unit (unitscale). 100 is the usual value; doubling it (200) makes the scene twice as large. A value <=0 leaves the current scale unchanged.

Returns: The number of triangles loaded (0 if loading fails).

Note: Loading is BLOCKING: with very large scenes the watchdog exception 142 ("process seems stuck, waiting for FRAME") may trigger. If it does, you can suppress it with ignore_error(142).
Note: Changing the map with LOAD_WLD9 does NOT require stopping mode 9: it frees the old mesh and materials and rebuilds the collision data without touching the camera.
Example program: 
// Load the castle courtyard (material 0 = CASTLE.MAP)
ntris = LOAD_WLD9("CASTLE.OBJ", "MAP/CASTLE.MAP", 100);
if (ntris == 0)
    say("Could not load the scene");
end

// ... later, when crossing a door, hot-swap the map:
LOAD_WLD9("CASTIN.OBJ", "MAP/CASTIN.MAP", 100);
x = 0; y = -300; angle = -90000;   // reposition the player in the new map

The return value (number of triangles) lets you verify that loading succeeded: 0 means the file does not exist or is invalid.
