MODE9_BILLBOARD()

MODE9_BILLBOARD( <id> , <wx> , <wy> , <wz> , <base size> )

Description:

Places a process inside the 3D world as if it were a cardboard cut-out: a flat drawing that always faces the camera and appears larger or smaller depending on how near or far it is. This is called a billboard, and it is the classic technique (the same as mode 7) for putting characters, enemies, trees or torches into a 3D scene without modelling them in 3D.

You tell it where in the world the sprite goes (wx, wy, wz) and the function calculates which part of the screen it falls on and what size it should appear at, writing those values into the process fields. It returns 1 if the sprite is visible and 0 if it is not (because it is behind the camera or occluded by a wall).

Multi-view sprites: just like in mode 7, you can make a character look different depending on the viewing angle (front, back, side...). For this the process must have a table of graphics in its xgraph field and its orientation in angle; mode 9 automatically picks the appropriate drawing based on the angle from which the camera views it.

Parameters:

ParameterMeaning
idThe process you want to display as a sprite in the 3D world.
wx, wy, wzWhere to place it in the world (wx, wy on the ground plane; wz = height), in game units.
base sizeThe sprite size at normal distance (same as the size field). Mode 9 scales it up or down according to distance.

Returns: 1 if the sprite is visible, 0 if it is hidden (behind the camera or occluded).

Note: For stationary sprites (a torch, a tree) there is a more convenient automatic way: simply set ctype = c_m9 in the process and leave its world coordinates in x, y, z; mode 9 draws it automatically each frame without any explicit call (see MODE9_CTYPE()).
Example program: 
PROCESS torch(wx, wy, wz)
BEGIN
    graph = 11;
    LOOP
        // project manually each frame
        if (MODE9_BILLBOARD(id, wx, wy, wz, 100))
            // visible: DLL has already set x, y, size, z
        end
        FRAME;
    END
END

If you prefer not to call it every frame, assign ctype = c_m9 to the process and keep x, y, z in world coordinates: mode 9 projects it automatically.
