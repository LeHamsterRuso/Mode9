MODE9_BIND()

MODE9_BIND( <OFFSET m9.z> )

Description:

Tells mode 9 WHICH m9 structure to read the camera height and tilt from each frame. "Binding" simply connects your m9 variable to the DLL so that whenever you change m9.height or m9.angle in your program, the camera reacts immediately.

In practice you almost never need to call this: START_MODE9() already performs this binding via its second parameter. It is only useful in one specific case: when you have multiple cameras, each with its own m9 structure, and you want to switch between them without stopping and restarting the render.

Parameters:

ParameterMeaning
OFFSET m9.zThe m9 structure you want mode 9 to use. Always pass its first field with OFFSET m9.z (or &m9.z).

Returns: 0.

Example program: 
// Two cameras: the player's and a security camera.
// Each stores ITS OWN height and tilt in its own structure.
GLOBAL
    STRUCT m9_player  z; camera; height; angle;  END
    STRUCT m9_seccam  z; camera; height; angle;  END
BEGIN
    ...
    // Press TAB to switch to the security camera view
    if (key(_tab))
        MODE9_BIND(OFFSET m9_seccam.z);   // 3D now uses the other camera
    end
    if (key(_space))
        MODE9_BIND(OFFSET m9_player.z);  // back to player view
    end
END

If your game has a single camera (the normal case), forget this function: the second parameter of START_MODE9() is enough.
