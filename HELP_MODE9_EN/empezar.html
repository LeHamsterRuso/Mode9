Getting started

This guide builds a mode 9 game in stages, from the simplest case to the most complete. Each stage adds just one new thing on top of the previous one, so you can stop wherever you like and still have something that works. It follows the same order as the MODE9_01 to MODE9_05 examples that ship with the DLL.

The bare minimum you need: a scene as an .OBJ file (triangulated, Y up) and its texture as a .MAP. If you don't have one yet, use the example's (CASTLE.OBJ + MAP/CASTLE.MAP). To prepare your own models, see OBJ Models.

Stage 1 · An empty world to walk around

The skeleton of every mode 9 game. You load the scene with LOAD_WLD9(), place the camera with MODE9_SET() and start the render with START_MODE9(). From then on, moving the camera is like moving any DIV process: advance() to move forward and change angle to turn.

The essentials: 
PROGRAM stage1;
IMPORT "MODE9.DLL";
CONST
    c_m9 = 200;
GLOBAL
    STRUCT m9   z; camera; height; angle;   END
BEGIN
    set_mode(320200);
    set_fps(60, 0);

    LOAD_WLD9("CASTLE.OBJ", "MAP/CASTLE.MAP", 100);  // scene
    LOAD_SKY9("MAP/SKY.MAP");                     // sky (optional)
    MODE9_STEP(70);                              // climb steps

    MODE9_SET(70, 0);                            // eye height, pitch
    START_MODE9(id, OFFSET m9.z, 0);         // render!

    LOOP
        if (key(_w)) advance(20); end
        if (key(_s)) advance(-20); end
        if (key(_left))  angle += 4000; end
        if (key(_right)) angle -= 4000; end
        if (key(_esc)) break; end
        FRAME;
    END
    STOP_MODE9();
END

See: LOAD_WLD9() · START_MODE9() · MODE9_SET() · MODE9_STEP() · MODE9_COLLISION()

Stage 2 · Populate it with billboards (2D sprites)

The quickest things to put in the world are billboards: flat 2D drawings that always face the camera and scale with distance, just like in mode 7. They work for trees, torches, items and even simple enemies. The most convenient way is the automatic one: give the process ctype = c_m9 and its world coordinates in x, y, z, and mode 9 draws it for you every frame without you calling anything.

An automatic billboard process: 
// In the main BEGIN, after START_MODE9, load the FPG and spawn props:
load_fpg("FPG/MODE9.FPG");
torch(300, -500, 0);
torch(-300, -500, 0);
...
PROCESS torch(x, y, z)
BEGIN
    ctype = c_m9;          // <- marks it as a mode 9 billboard
    graph = 11;            // the torch drawing in the FPG
    size  = 100;           // base size (distance does the rest)
    LOOP
        // x, y, z are WORLD coordinates; don't touch anything else.
        FRAME;
    END
END

See: MODE9_BILLBOARD() · MODE9_CTYPE() · MODE9_OCCLUDE() · MODE9_PROJECT()

Stage 3 · Add real 3D objects

When a flat billboard is not enough (you want to walk around the object and see it from every side), use a 3D model. You load it once with LOAD_OBJ9(), which returns a handle, and draw it every frame with DRAW_OBJ9() wherever you want. The same handle can be drawn several times to repeat the object.

Load and draw a 3D model: 
GLOBAL
    h_cat;
...
    h_cat = LOAD_OBJ9("OBJ/CAT.OBJ", "MAP/CAT.MAP", 50);
    LOOP
        // drawn at (x, y, z) facing a given angle, every frame
        DRAW_OBJ9(h_cat, 400, -1300, 0, 0);
        // ... move the camera, etc.
        FRAME;
    END
Background loading: LOAD_OBJ9 returns the handle instantly, but the file takes a few frames to be ready. That's fine: DRAW_OBJ9 ignores the model until it loads, so it just pops in. To wait for it, use MODE9_OBJ9_READY().

See: LOAD_OBJ9() · DRAW_OBJ9() · MODE9_OBJ9_READY()

Stage 4 · Animate a 3D object

To make the object move (a breathing cat, a walking enemy), load it with LOAD_OBJ9_ANIM() instead of LOAD_OBJ9. The animation is several poses stored as numbered files (CAT0.OBJ, CAT1.OBJ...), and mode 9 blends the in-between poses for you so it looks smooth. You draw it with DRAW_OBJ9_ANIM(), which takes one extra parameter, fpos, telling how far into the animation it is.

Load an animation and play it in a loop: 
CONST
    NFRAMES = 10;      // CAT0.OBJ .. CAT9.OBJ
    LOOP_CS = 300;     // one loop lasts 3 s (300 hundredths)
GLOBAL
    h_cat;
...
    h_cat = LOAD_OBJ9_ANIM("OBJ/CAT", "MAP/CAT.MAP", 50, NFRAMES);
    LOOP
        // fpos tied to the clock: same speed whatever FPS the game runs at
        DRAW_OBJ9_ANIM(h_cat, 400, -1300, 0, 0,
                       (timer[0] % LOOP_CS) * (NFRAMES*256) / LOOP_CS);
        FRAME;
    END

See: LOAD_OBJ9_ANIM() · DRAW_OBJ9_ANIM()

Stage 5 · Move from one map to another

A big level is split into several maps, loading the next one as you go through a door. The key: LOAD_WLD9() reloads the scene HOT, without stopping mode 9 or losing the camera. The trick to hide the jump is a fade to black while the mesh is swapped and the player is repositioned.

Going through a door into the interior: 
// Is the player right by the door? (squared distance < radius²)
if ((x-DOOR_X)*(x-DOOR_X) + (y-DOOR_Y)*(y-DOOR_Y) < RADIUS2)
    fade_off(); frame; fade_on(); frame;   // fade to black

    signal(type yard_props, s_kill);      // remove the old map's props

    LOAD_WLD9("OBJ/CASTIN.OBJ", "MAP/CASTIN.MAP", 100);  // new map
    SKY9_OFF();                              // interior: no sky
    SET_ENV_COLOR9(8, 8, 12);           // dark ambience

    x = 0; y = -300; angle = -90000;   // reposition the player
end
Watch out for re-triggering the door: after a swap, leave a few frames of margin (a "cooldown") before checking doors again, or you'll go in and out non-stop. And since LOAD_WLD9 is blocking, keep ignore_error(142) handy in case the watchdog fires on real hardware.

See: LOAD_WLD9() · SKY9_OFF() · SET_ENV_COLOR9() · STOP_MODE9()

From here on, it's all polish: fog and view distance (SET_FOG9(), MODE9_FARCLIP()), camera angle (MODE9_FOV()), split-screen for two players (MODE9_SPLIT()) or more speed on modest machines (MODE9_DOWNSCALE()). It's all in Functions.
