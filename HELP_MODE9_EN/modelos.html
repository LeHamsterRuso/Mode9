How to prepare OBJ models

Both the scene (LOAD_WLD9()) and 3D objects (LOAD_OBJ9()) are loaded from .OBJ files, the mesh format exported by modelling applications such as Blender, Wings3D or MilkShape. For mode 9 to read them correctly, they must be exported with specific settings.

Export settings

SettingValueWhy
FacesTriangulated The engine only draws triangles. If the model has quads or n-gons, triangulate it before exporting (in Blender: Triangulate Faces, or tick Triangulated Mesh in the export dialog).
Forward axis-Z Forward axis set to -Z, so the model's front aligns with the world's front.
Up axisY Vertical axis in Y (Y up). This is Blender's default, so normally nothing needs to be changed.
UVsIncluded (if textured) If the model is textured, export the UV coordinates. Without UVs the model is drawn with single-colour flat-shading.

With those axes (Forward -Z, Up Y) the model fits directly into mode 9's world coordinate system, where Y is the height axis.

Materials and textures

A scene can have several materials declared with usemtl in the OBJ. They are numbered in order of appearance (0, 1, 2...) and each is assigned a texture with MODE9_MATERIAL() (material 0 is already set by LOAD_WLD9()).

LimitValueNotes
Materials per scene16 Numbered 0 to 15, in the order of the usemtl entries.
Texture size per material1024 x 1024 px maximum Textures are DIV .MAP files (palette-indexed to the game palette). Common sizes are powers of two: 128, 256, 512 or 1024.
Watch out for file names: mode 9 runs on DOS, so the file name (.OBJ and .MAP) must have 8 characters or fewer (not counting the extension). For example CASTIN.OBJ is valid, but CASTLE_IN.OBJ (9 letters) would fail to load silently: the model would simply appear empty. If a model does not appear, the first thing to check is the filename length.

Workflow summary

Example program: 
// 1) In Blender: triangulate the mesh and export to OBJ with
//    Forward axis = -Z, Up axis = Y, and include UVs.
//    Save the .OBJ with a name of 8 letters or fewer: CASTLE.OBJ

// 2) Convert textures to .MAP (1024x1024 maximum) with DIV.

// 3) In the PRG, load everything:
LOAD_WLD9("CASTLE.OBJ", "MAP/CASTLE.MAP", 100);  // mesh + material 0
MODE9_MATERIAL(1, "MAP/STAIRS.MAP");             // material 1 (usemtl)
MODE9_MATERIAL(2, "MAP/WATER.MAP");              // material 2 (usemtl)
