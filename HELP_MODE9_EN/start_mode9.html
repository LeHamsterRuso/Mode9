START_MODE9()

START_MODE9( <camera id> , <OFFSET m9.z> , <region> )

Description:

Activates the 3D scene render for mode 9. It is the equivalent of start_mode8(), with the same three-parameter signature. From this call on, the DIV2 BACKGROUND layer is no longer a 2D graphic but the three-dimensional scene that the DLL draws every frame.

Before calling it you must load the scene with LOAD_WLD9(). The camera is placed at the position of the indicated process (its x, y fields on the ground plane and angle as horizontal rotation), at the height and with the tilt defined in the m9 structure.

Parameters:

ParameterMeaning
camera idIdentifier of the process whose position the camera will use. Normally id (the process that controls the game itself).
OFFSET m9.zAddress of the m9 structure (pass &m9.z, its first field). The DLL reads m9.height (eye height) and m9.angle (pitch) from there every frame. Equivalent to binding the structure.
regionScreen region: 0 = full screen; 1..15 = a region previously defined with MODE9_REGION() (multiple calls create a split-screen with regions, like mode 8 viewports).

Returns: 1 (always, when activated).

Note: A DLL cannot create STRUCTs or c_types in the DIV2 language, which is why the structure m9 (fields z, camera, height, angle) and the constant c_m9 are declared in the PRG and passed to the DLL via this function and MODE9_SET().
Example program: 
PROGRAM mode9_minimal;
IMPORT "MODE9.DLL";
CONST
    c_m9 = 200;
GLOBAL
    STRUCT m9
        z; camera; height; angle;
    END
BEGIN
    set_mode(320200);
    set_fps(60, 0);
    load_fpg("FPG/MODE9.FPG");
    // Load the scene: mesh + texture + scale
    LOAD_WLD9("CASTLE.OBJ", "MAP/CASTLE.MAP", 100);
    // Place the camera 70 units high
    MODE9_SET(70, 0);
    // Activate the render on full screen
    START_MODE9(id, OFFSET m9.z, 0);
    LOOP
        if (key(_up))   advance(20);  end
        if (key(_down)) advance(-20); end
        if (key(_left))  angle += 4000; end
        if (key(_right)) angle -= 4000; end
        FRAME;
    END
END

This program loads a 3D scene and lets you walk through it. Notice that moving the camera is just as simple as in a normal 2D game: advance() moves it forward and changing the process angle makes it turn, exactly as you would with any process — only now what you see is the 3D world from its position.
