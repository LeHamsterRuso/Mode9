Mode 9 for DIV Games Studio 2

Mode 9 (the library MODE9.DLL) is an extension for DIV Games Studio 2 that lets you display and walk through a three-dimensional scene loaded from an .OBJ file (the mesh format exported by programs such as Blender). Unlike a flat background, the scene has real volume: walls, floors, ceilings and ramps at any angle, and you can move freely inside it.

It is the big brother of mode 8: there the map was built with flat sectors (.WLD); here the map is any 3D mesh modelled in a 3D application. The camera is a normal DIV process that you place wherever you want and point in any direction. The most common setup is to put it at the player's eyes (first-person view, as in the examples), but nothing forces you to do so: it also works as a follow camera behind the character, a top-down view, or fixed cameras.

On top of that 3D scene you can draw your characters and objects in two ways: as flat sprites (2D drawings that scale with distance and always face the camera, just like mode 7) or as real 3D models, with smooth animation.

Basic concepts

Moving the camera is straightforward: since it is a normal process, you advance it with advance(), strafe with xadvance(), rotate by changing its angle field, and move up or down with the eye height (m9.height). What you see on screen is the world from wherever that process is.

World coordinate system (DIV2 units, divided by unitscale to get OBJ units):

AxisMeaning
X = process.xGround plane (east / west).
Z = process.yGround plane (north / south).
Y = process.z + m9.heightVertical axis (height, Y up as in Blender / OBJ).
yaw = process.angleHorizontal rotation, in thousandths of a degree (180000 = 180 degrees).
pitch = m9.angleVertical tilt, in thousandths of a degree (0 = horizon).
About m9 and c_m9: a DLL cannot create STRUCTs or c_types in the DIV2 language (those are generated by the compiler). That is why the structure m9 (fields z, camera, height, angle, the same as m8) and the constant c_m9 = 200 are declared in the PRG and passed to the DLL via START_MODE9() and MODE9_SET(). Billboard processes use ctype = c_m9.

A minimal program

Example program: 
PROGRAM mode9_demo;
IMPORT "MODE9.DLL";
CONST
    c_m9 = 200;           // billboard logical ctype (must match the value inside the DLL)
GLOBAL
    STRUCT m9             // same shape as m8: z, camera, height, angle
        z; camera; height; angle;
    END
    scr_x; scr_y;
BEGIN
    set_mode(320200);
    set_fps(60, 0);
    scr_x = MODE9_SCRW();
    scr_y = MODE9_SCRH();

    // 1) Load the scene (mesh + texture + scale) in one call
    LOAD_WLD9("OBJ/CASTLE.OBJ", "MAP/CASTLE.MAP", 100);
    LOAD_SKY9("MAP/SKY.MAP");      // panoramic sky
    MODE9_STEP(70);                // climb stairs up to 70 units high

    // 2) Spawn the camera process (sets m9.camera, m9.height and starts the render)
    camara();
END

PROCESS camara()
BEGIN
    x = 0;  y = -1500;  z = 0;
    angle = -90000;
    m9.camera = id;              // REQUIRED: DLL reads this every frame
    m9.height = 160;             // eye height
    m9.angle  = 0;               // pitch (0 = level horizon)
    START_MODE9(id, OFFSET m9.z, 0);

    // 3) Game loop: move the camera
    LOOP
        if (key(_w)) advance(20);  end
        if (key(_s)) advance(-20); end
        angle += (scr_x/2 - mouse.x) * 200;   // rotate with the mouse
        if (key(_esc)) STOP_MODE9(); break; end
        FRAME;
    END
END
