MODE9_PROJECT()

MODE9_PROJECT( <wx> , <wy> , <wz> , <OFFSET sx> , <OFFSET sy> , <OFFSET sz> )

Description:

Answers the question: "where on screen does this 3D world point appear?". You give it a world position (wx, wy, wz) and it returns the screen coordinates (sx, sy) where it would be seen, plus its depth (sz, how far away it is). It is like MODE9_BILLBOARD(), but instead of moving a process it hands you the numbers so you can do whatever you want with them.

This is exactly what you need to anchor 2D information to a world object: an NPC's name floating above their head, a speech bubble, an arrow pointing to a target, or a health bar. It returns 0 if the point is behind the camera (in which case nothing should be drawn).

Parameters:

ParameterMeaning
wx, wy, wzWorld point to project (DIV2 units).
OFFSET sxVariable that receives the screen X coordinate (pass OFFSET sx).
OFFSET syVariable that receives the screen Y coordinate.
OFFSET szVariable that receives the depth (z) of the point.

Returns: 1 if the point is visible, 0 if it is hidden.

Example program: 
PRIVATE
    nsx; nsy; nsz;
END
    // Anchor the NPC name above their head
    if (MODE9_PROJECT(0, 500, 220, OFFSET nsx, OFFSET nsy, OFFSET nsz))
        // visible on screen: draw the label at (nsx, nsy)
        write(0, nsx, nsy, 4, "Nazuma");
    end

Because it writes to variables rather than moving a process, it is ideal for world-anchored HUD elements (NPC labels, speech bubbles, objective icons).
